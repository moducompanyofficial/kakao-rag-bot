# 회사 문서로 답하는 AI 상담봇 (RAG + 카카오톡)

회사 문서(.md/.txt)를 학습해 **출처를 인용하며** 답하는 상담봇. 웹 위젯과 카카오 i 오픈빌더 스킬서버를 동시 지원. 크몽 상품 2("AI 상담봇 구축")의 쇼케이스 데모이자 실제 납품 베이스 코드.

## 구조

```
docs/*.md ──ingest──▶ data/store.json (청크+임베딩)
                              │
사용자 질문 ──embed──▶ topK 검색 ──▶ Claude (근거 제한 프롬프트) ──▶ 출처 인용 답변
    ▲                                                                  │
    ├── 웹 데모 (public/index.html → POST /api/chat) ◀─────────────────┤
    └── 카카오톡 (오픈빌더 스킬 → POST /kakao/skill, 콜백 모드 지원) ◀──┘
```

설계 원칙: 문서에 없으면 답하지 않고 상담원 연결 안내 (환각 통제). 가격·환불·법적 책임은 문서 명시 시에만 답변.

## 실행 (비용 0원 — 무료 티어)

```bash
cp .env.example .env   # GEMINI_API_KEY 하나만 입력
npm install
npm run ingest -- ./docs   # 샘플 FAQ 학습
npm start                  # http://localhost:3000
```

API 키: https://aistudio.google.com/apikey 에서 무료 발급 (구글 계정만 필요, 카드 불필요). Gemini 무료 티어가 임베딩+생성을 모두 처리한다. 무료 티어 레이트 리밋(분당 요청 수 제한)은 데모 트래픽에 충분하며, 납품 시에는 .env에 Anthropic/OpenAI 키를 넣으면 자동으로 유료 스택으로 전환된다(코드 수정 불필요).

## 카카오톡 채널 연동 (납품 시)

1. 카카오 비즈니스 → 채널 생성 → 카카오 i 오픈빌더 봇 연결
2. 폴백 블록에 스킬 연결: `https://<배포 도메인>/kakao/skill`
3. **오픈빌더 타임아웃 5초** — 운영 시 "AI 챗봇 전환" 승인 후 콜백 모드 사용 (코드는 콜백 자동 감지, `src/channels/kakao.js`)

## 운영 전환 체크리스트 (데모 → 납품)

- 벡터 스토어: `src/rag/store.js` → pgvector/Qdrant 교체 (인터페이스 동일)
- 임베딩: OpenAI 불가 고객사는 `src/rag/embed.js`만 교체 (Upstage 솔라 등)
- 대화 로그 DB 적재 + 미답변 질문 리포트 (월 운영 계약의 핵심 산출물)
- 관리자 데이터 갱신 UI, 레이트 리밋, 채널별 인증

## 데모 시나리오 (영업용)

1. "배송은 얼마나 걸리나요?" → 문서 근거 + 출처 인용 확인
2. "VVIP 혜택 알려주세요" → 표 데이터 정확 인용 확인
3. "사장님 연봉이 얼마예요?" → 환각 없이 상담원 연결 안내 (통제력 시연 — 이 응답이 계약을 만든다)
